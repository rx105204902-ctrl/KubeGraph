## ADDED Requirements（新增需求）

### Requirement: 系统必须规范化图谱渲染输入并隔离异常元素
系统必须在图谱进入渲染层之前，对节点和边执行稳定 ID 去重、引用校验和异常隔离；任何单个异常元素都不得导致整个图谱无法显示。

#### Scenario: 图数据包含重复节点
- **WHEN（当）** 图谱载荷中出现多个相同节点 ID 的元素时
- **THEN（则）** 系统只保留一个规范化后的节点实例，并继续渲染与其关联的有效边

#### Scenario: 图数据包含无效边
- **WHEN（当）** 某条边指向不存在的 `source` 或 `target` 节点时
- **THEN（则）** 系统忽略该异常边、记录诊断信息，并继续渲染其余有效图元素

### Requirement: 系统必须在图谱更新时保持稳定视图
系统必须在筛选、扩展、折叠、详情加载或局部数据更新时，尽量复用现有图元素、保留当前视口和选中状态，并避免不必要的全量清空与重建。

#### Scenario: 用户切换边类型筛选
- **WHEN（当）** 用户调整边类型筛选条件时
- **THEN（则）** 系统必须只更新可见元素与高亮状态，而不是重置整张图的视口与当前选中节点

#### Scenario: 用户展开一个子图
- **WHEN（当）** 用户为已选节点展开新的邻居关系时
- **THEN（则）** 系统必须批量合并新增元素，并在保留原有上下文的前提下完成必要的局部或受控重布局

### Requirement: 系统必须在大规模图谱下采用性能保护策略
系统必须在节点或边数量超过阈值时，切换到低成本布局与渲染策略，避免出现长时间卡顿、页面无响应或浏览器崩溃。

#### Scenario: 图谱元素超过性能阈值
- **WHEN（当）** 当前可见节点或边数量超过预设性能阈值时
- **THEN（则）** 系统必须关闭高成本动画或装饰效果，并采用更轻量、可预测的布局方式

#### Scenario: 布局计算超时或失败
- **WHEN（当）** 当前布局在限定时间内未完成，或布局引擎执行失败时
- **THEN（则）** 系统必须回退到备用布局策略，并向用户提示已应用性能保护或回退行为

### Requirement: 系统必须为渲染失败和无数据场景提供恢复路径
系统必须在图谱请求失败、布局失败或结果为空时，展示明确且可操作的反馈，而不是留给用户空白画布或纯技术报错。

#### Scenario: 图谱加载失败
- **WHEN（当）** 图谱数据请求失败或规范化过程失败时
- **THEN（则）** 系统必须展示友好的失败提示，并提供重试或重置视图的操作

#### Scenario: 当前查询没有图结果
- **WHEN（当）** 用户的搜索或筛选未返回任何节点与边时
- **THEN（则）** 系统必须展示空状态说明，并告知用户下一步可以调整的条件
