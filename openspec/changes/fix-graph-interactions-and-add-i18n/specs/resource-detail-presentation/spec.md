## ADDED Requirements（新增需求）

### Requirement: 系统必须统一按钮视觉层级和交互状态
系统必须为主按钮、次按钮和文本按钮提供统一的视觉规范，并保证 hover、active 和 disabled 状态清晰可辨。

#### Scenario: 用户查看主要操作按钮
- **WHEN（当）** 用户浏览界面中的主要操作按钮时
- **THEN（则）** 系统必须通过颜色、阴影或层次明确区分主要操作和次要操作

#### Scenario: 用户悬停或点击按钮
- **WHEN（当）** 用户悬停、按下或禁用按钮时
- **THEN（则）** 系统必须提供一致的状态反馈，而不是不同按钮表现割裂

### Requirement: 系统必须将右侧资源 metadata 结构化展示为表格
系统必须优先将右侧资源面板中的 `metadata` 以结构化表格形式展示，确保字段名和值清晰对齐，提升阅读效率。

#### Scenario: 用户查看资源 metadata
- **WHEN（当）** 用户打开某个资源的详情面板时
- **THEN（则）** 系统必须以表格或等效结构化视图展示主要 metadata 字段，而不是仅输出整段 JSON

#### Scenario: metadata 字段较多
- **WHEN（当）** 当前资源的 metadata 字段较多时
- **THEN（则）** 系统必须保持字段对齐和可读性，不因字段长度不同而造成明显错乱

### Requirement: 系统必须为 metadata 展示保留扩展能力
系统必须让 metadata 展示结构具备后续增加字段、排序或筛选能力的扩展空间。

#### Scenario: 未来需要增加字段或排序能力
- **WHEN（当）** 后续版本需要扩展 metadata 字段展示或引入排序/筛选行为时
- **THEN（则）** 当前结构必须支持扩展，而不需要彻底重写详情面板展示方式
